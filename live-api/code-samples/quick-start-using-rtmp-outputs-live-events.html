    Inicio rápido: Uso de salidas RTMP para eventos en directo

    Este tutorial le guiará a través de la creación de salidas RTMP para configurar eventos en vivo en YouTube Live y Facebook Live.

    Resumen

    Este tutorial explica a nivel de API cómo crear un trabajo de transmisión en vivo en Brightcove en vivo y, a continuación, crear salidas RTMP y volver a transmitirlas a YouTube Live y Facebook Live.

    Notas:
    • Para obtener información sobre cómo utilizar salidas RTMP en el módulo Live, consulte Uso de salidas RTMP en vivo para transmitir a Facebook y YouTube.
    • Salidas RTMP hace apoyo rtmps urls - debes usar rtmps si la plataforma social lo admite.
    • Tenga en cuenta que Facebook finalizó el soporte para rtmp.streams a partir del 1 de mayo de 2019, para Facebook, DEBE usar RTMPS.
    • El único formato actual compatible con las URL RTMP es.rtmps://host:port/application/streamName

    Este tutorial utilizará para hacer las solicitudes API, pero puede hacerlas fácilmente en, u otros clientes REST en su lugar.curlInsomniaPostman

    Crear un trabajo en vivo

    Primero, crearemos un trabajo en vivo. Necesitará una para la API en vivo.API-KEY Si no lo hace, y está interesado en obtener acceso, póngase en contacto con su gestor de cuenta.

    1. Copie y pegue el siguiente curl comando en un editor de texto:

      curl -X POST \
  https://api.bcovlive.io/v1/jobs \
  -H 'Content-Type: application/json' \
  -H 'x-api-key: ' \
  -d '{
    "live_stream": true,
    "region": "",
    "outputs": [{
      "label": "hls360p",
      "live_stream": true,
      "height": 360,
      "video_bitrate": 365,
      "segment_seconds": 6,
      "keyframe_interval": 60
    },
    {
      "label": "hls432p",
      "live_stream": true,
      "height": 432,
      "video_bitrate": 730,
      "segment_seconds": 6,
      "keyframe_interval": 60
    },
    {
      "label": "hls540p",
      "live_stream": true,
      "height": 540,
      "video_bitrate": 2000,
      "segment_seconds": 6,
      "keyframe_interval": 60
    },
    {
      "label": "hls720p3M",
      "live_stream": true,
      "height": 540,
      "video_bitrate": 2000,
      "segment_seconds": 6,
      "keyframe_interval": 60
    },
    {
      "label": "hls720p4.5M",
      "live_stream": true,
      "height": 540,
      "video_bitrate": 4500,
      "segment_seconds": 6,
      "keyframe_interval": 60
    }]
  }'

    2. Reemplazar:

      • con la región disponible más cercana a su codificador, consulte las regiones disponibles más cercanas a su codificador. Por ejemplo: us-west-2
      • con su clave API en vivo de Brightcove. Por ejemplo: abcdfeg-this-is-a-fake-api-key-FgJajjasd12hJHsZ
    3. Después de realizar esos cambios, copie y pegue el código en Terminal o cualquier aplicación de línea de comandos que use y ejecútelo.

    4. La respuesta debería ser algo como esto:

      {
    "id": "3b6871bf2f344acaa6b397d09b476018",
    "outputs": [... removed for simplicity ...],
    "stream_url": "rtmp://ep1-usw2.bcovlive.io:1935/3b6871bf2f344acaa6b397d09b476018",
    "stream_name": "alive",
    "static": false,
    "encryption": {},
    "playback_url": "https://bcovlive-a.akamaihd.net/3b6871bf2f344acaa6b397d09b476018/us-west-2/NA/playlist.m3u8",
    "playback_url_dvr": "https://bcovlive-a.akamaihd.net/3b6871bf2f344acaa6b397d09b476018/us-west-2/NA/playlist_dvr.m3u8"
}

    Estos trabajos crearán 5 copias basadas en las recomendaciones de Apple.

    Configurar el codificador

    Los pasos que se muestran aquí asumirán un codificador Elemental Live Box, que se utilizó para probar los pasos de este tutorial. Si tiene un codificador diferente, la configuración debería ser similar.

    Los ajustes más importantes para este caso son:

    • Configurar la fuente del código de tiempo como «reloj del sistema»
    • Establezca «Frecuencia de código de tiempo OnFi» en 1
    • Marque «inserción de código de tiempo» dentro del vídeo
    • Enérgicamente recomendable: Velocidad de fotogramas = seguir el origen

    Añadir la transmisión en directo a YouTube en vivo

    Asumimos aquí que tienes una cuenta de YouTube habilitada para la transmisión en vivo.

    1. Haga clic en ir en vivo, y usted debería ver:

    2. Haga clic en «Encoder streaming en vivo»:

    3. Configure los metadatos de YouTube del evento en vivo y utilice los datos «Configuración del codificador» para conectar la transmisión en vivo de Brightcove a YouTube:

    4. Ahora que tenemos YouTube Live configurado, crearemos la salida RTMP para ella desde nuestro trabajo en vivo.

    5. Copie el siguiente curl comando en un editor de texto:

      curl -X POST \
  https://api.bcovlive.io/v1/jobs//rtmpouts \
  -H 'Content-Type: application/json' \
  -H 'x-api-key: ' \
  -d '{
  "url": "rtmp://a.rtmp.youtube.com/live2/"
}'

    6. Reemplazar:

      • con su ID de trabajo en vivo de Brightcove. En nuestro ejemplo, este valor sería 3b6871bf2f344acaa6b397d09b476018 - su valor, desde la respuesta a su solicitud de crear el trabajo en vivo, será diferente.
      • para su clave API en vivo de Brightcove. Por ejemplo: abcdfeg-this-is-a-fake-api-key-FgJajjasd12hJHsZ
      • para el nombre de la secuencia/clave que proporciona YouTube
    7. Pegue el código editado en su aplicación de línea de comandos y ejecútelo.

    8. La respuesta debería ser algo como esto:

      {
          "connection_info": {
            "host": "a.rtmp.youtube.com",
            "port": 1935,
            "application": "live2",
            "streamName": ""
          },
          "duration_history": [],
          "rtmp_out_id": "27aacdfa5bcf4436ade0519e6a748aba",
          "stream_start": 1542902218238
        }

    9. Para probar la reproducción de YouTube puedes ir a YouTube mi canal:

    Añadir la transmisión en directo a Facebook en vivo

    Estamos asumiendo aquí que tiene una cuenta de Facebook habilitada para la transmisión en vivo.

    1. Haga clic en vídeo en directo

    2. Haga clic en conectar y transmitir la clave para obtener los datos necesarios para conectar su trabajo en vivo de Brightcove:

    3. A continuación, cree una salida en vivo para el trabajo en vivo de Brightcove utilizando «url del servidor» y «clave de transmisión» de Facebook. Copie el siguiente curl comando y péguelo en un editor de texto:

      curl -X POST \
  https://api.bcovlive.io/v1/jobs//rtmpouts \
  -H 'Content-Type: application/json' \
  -H 'x-api-key: ' \
  -d '{
  "url": "rtmp://live-api-s.facebook.com:80/rtmp/"
}'

    4. Reemplazar:

      • con su ID de trabajo en vivo de Brightcove. En nuestro ejemplo, este valor sería 3b6871bf2f344acaa6b397d09b476018 - su valor, desde la respuesta a su solicitud de crear el trabajo en vivo, será diferente.
      • con su clave API en vivo de Brightcove. Por ejemplo: abcdfeg-this-is-a-fake-api-key-FgJajjasd12hJHsZ
      • con tu nombre de transmisión que Facebook te da

    5. La respuesta debería ser algo como esto:

      {
  "connection_info": {
      "host": "live-api-s.facebook.com",
      "port": 80,
      "application": "rtmp",
      "streamName": ""
  },
  "duration_history": [],
  "rtmp_out_id": "59323c3d1e2b44748755853193fb126c",
  "stream_start": 1542904101097
}

    6. Para probar su transmisión en vivo de Facebook, haga clic en «Go Live» en su página web de Facebook.
