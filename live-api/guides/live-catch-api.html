    API de puesta al día en vivo

    En este tema se explica cómo crear un trabajo activo con recuperación mediante la API Live Catch-up (TimeShift).

    Introducción

    La API Live Catch-up es una adición a la API Live que proporciona capacidad de reproducción para el inicio o parte de una transmisión en vivo para permitir que los espectadores tardíos se «pongan al día».

    La capacidad de recuperación se agrega a un trabajo activo a través de un nuevo campo de timeshift objeto utilizado al crear un trabajo activo. Los detalles de los campos de timeshift objeto se detallan en las secciones siguientes.

    Campos TimeShift

    El timeshift objeto es un objeto de nivel superior en el cuerpo de la solicitud para crear una solicitud de trabajo Live. En la siguiente tabla se detallan los campos de este objeto.

    timeshift Campos
    Campo Descripción
    startover_duration Tamaño de la ventana de inicio en segundos, hasta 604800 (7 días).

    Si timeshift.startover_duration se especifica en la solicitud, se creará una URL de reproducción habilitada para el cambio de tiempo para cada lista de reproducción. Estas URL admitirán la reproducción en vivo del borde (con un tamaño DVR según lo especificado por el live_duration parámetro).

    También admitirán la reproducción de porciones arbitrarias dentro del período definido por startover_duration añadiendo parámetros:

    • start=ts - Marca de tiempo de época Unix del inicio de la reproducción
    • end=ts - Marca de tiempo Unix época del final de reproducción
    live_duration Tamaño de la ventana activa en segundos hasta 300. El valor predeterminado es 4 x duración del segmento.
    live_delay Número de segundos para retrasar la salida en vivo. El mínimo y el valor predeterminado es de 5 segundos.
    type
    • single - para un trabajo de turno de tiempo independiente (predeterminado)
    • primario: para el trabajo de turno de tiempo primario (debe ser un trabajo SEP)
    • copia de seguridad: para un trabajo de cambio de tiempo de respaldo
    primary_job_id Hace referencia al identificador de trabajo de turno de tiempo principal (necesario si el tipo es copia de seguridad).
    skip_live_output No generar listas de reproducción normales en vivo o dvr (el valor predeterminado es falso)

    Notas

    • Tenga en cuenta que las URL de timeshift no se incluyen en la respuesta de la API si timeshift.type es «backup». Utilice las direcciones URL de oculto de horas indicadas para el trabajo principal.
    • Si se requiere DRM para una salida de cambio de tiempo, incluya un drm campo en la solicitud de trabajo como de costumbre. Este atributo sólo debe especificarse para el trabajo principal para proteger la salida del cambio de tiempo; sin embargo, puede ser conveniente establecer skip_live_output en true para cualquier trabajo de copia de seguridad para evitar la reproducción sin protección.
    • El recorte está limitado a las últimas 24 horas, y al igual que la redundancia en vivo, solo incluirá la entrada del trabajo seleccionado

    Obtener detalles de trabajo

    Ya sea que solicite una lista de trabajos o detalles para uno específico, se timeshift_url incluirán para cualquier lista de reproducción en un trabajo habilitado para el timeshift. La persona que llama puede agregar parámetros a esta URL para especificar las horas de inicio y parada como marcas de tiempo Unix Epoch. Por ejemplo:

    https://bcovlive-a.akamaihd.net/8d64308aafe34d17adb98af69e9eac13/us-east-1/6021289116001/ts-cf98fa7b2ee4450e-5df3aae6cb724fe1a4a932b4964d3e17/playlist_ts.m3u8?start=1593457200&end=1593460800

    Si el start y end los parámetros no están incluidos, la reproducción será en el borde en vivo.

    Crear token de reproducción

    Se han agregado nuevos campos al token de reproducción para permitir que la API Get Job Sources devuelva URL de desplazamiento de tiempo. Actualmente SSAI no es compatible con trabajos con cambio de tiempo habilitado, por lo que no se debe especificar un application_id atributo.

    Si ts_start y ts_end se especifican, se agregarán a la URL de reproducción como parámetros de consulta como se describe en la sección anterior.

    Nuevos campos

    Nuevos campos de token de reproducción
    Campo Descripción
    ts Seleccione la URL de reproducción de cambio de tiempo (true|false, opcional, el valor predeterminado es falso)
    ts_start Marca de tiempo de época Unix para el inicio de la ventana de inicio (número, opcional)
    ts_end Marca de tiempo de época Unix para el final de la ventana de inicio (número, opcional)
