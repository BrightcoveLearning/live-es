    API en vivo: Creación de clips VOD

    En este tema se muestra cómo crear clips de vídeo bajo demanda (VOD) a partir de sus transmisiones en directo.

    Resumen

    Los clips son vídeos extraídos de una transmisión en directo. Se pueden enviar a un depósito de S3, a un sitio FTP o a una Video Cloud cuenta. El clip se crea como un vídeo MP4, y eso es lo que se envía al destino en todos los casos. En el caso de Video Cloud, el MP4 será transcodificado por el sistema de ingesta, y qué tipos de copias se crean para el video dependerá del perfil de ingesta utilizado.

    Las definiciones de los clips se crean utilizando el /vods punto final.

    Los clips se pueden crear de varias maneras:

    • Con stream_start_timecode y/o stream_end_timecode definidos en códigos de tiempo SMPTE para el evento de transmisión en vivo - tenga en cuenta que esto requiere que el codificador envíe información de código de tiempo
    • Con start_time y/o end_time definido en relación con la hora de inicio ( stream_start_time ) de todo el evento de transmisión en vivo
    • Con start_time y/o end_time definido en tiempo de Época (Unix) (en segundos)
    • Con duration
    • La API VOD no se puede utilizar con trabajos cifrados .

    Notas

    1. Los perfiles de ingesta heredados ya no admiten recortes en vivo. Para crear un clip de vídeo a partir de una transmisión en directo, asegúrate de que tu cuenta esté habilitada para Entrega dinámica y utiliza un perfil de ingesta de entrega dinámica.
    2. Para que los clips estén disponibles lo más rápido posible, primero se crea un clip con precisión de segmentos y, a continuación, se reemplaza por un clip con precisión de marco tan pronto como esté disponible.
    3. Si especifica duration, el clip resultante será el siguiente:
      • Si el trabajo está activo y sigue activo: (tiempo de solicitud - duración) a (tiempo de solicitud)
      • Si el trabajo ha finalizado: ( finished_at - duración) a ( finished_at)
    4. Si especifica tanto start_time AND end_time:
      • Si el trabajo está activo y sigue activo: siempre que la ventana de tiempo de Época se encuentre completamente dentro created_at y el tiempo de solicitud, el clip se realizará
      • Si el trabajo ha finalizado: siempre y cuando la ventana de tiempo de Época se encuentre completamente dentro created_at y finished_at , el clip se hará
    5. Los clips de transmisiones en directo que utilicen SSAI no incluirán anuncios.
    6. Los clips se pueden crear hasta 7 días después de un evento. Para SEP, se pueden crear hasta la próxima activación o 7 días (lo que sea más corto).
    7. La API VOD no agregará ningún contenido fuera de lo que está presente en la secuencia. Si especifica 350 en una transmisión en vivo de 300 segundos de duración, la salida tendrá 300 segundos de duración.
    8. No es necesario utilizar una transmisión en vivo habilitada para DVR para que el recorte funcione, ya que la transmisión en vivo se almacena mientras se transmite y está disponible inmediatamente y durante 7 días después de que finalizó el evento.
    9. El recorte de Brightcove Live solo producirá un clip con la misma resolución que la salida de resolución más alta. No coincidirá con la resolución de entrada de origen (a menos que sea la misma que la salida de resolución más alta).

    Los clips también se pueden enviar a varios destinos:

    • Una Video Cloud cuenta
    • Un servidor FTP
    • Un cubo S3

    Cuando especifica un clip, la salida debe Contiene ya sea un url destino o un videocloud objeto para detallar la creación del video y la ingestión del clip en Video Cloud.

    Nota: los clips se pueden crear mientras se ejecuta la transmisión en directo. Para ello, deberá definir las horas de inicio y finalización del clip en tiempo de época o relativa a la hora de inicio de la transmisión en vivo.

    Credenciales

    Si el destino al que va a enviar el clip requiere credenciales para acceder, puede crearlas utilizando las operaciones de credenciales de Live API. Consulte Administración de credenciales para Live API para obtener más detalles.

    Tenga en cuenta que si va a enviar el clip a una cuenta de Video Cloud asociada a la cuenta Live, no debe incluir las credenciales en el trabajo de recorte.

    Endpoint

    Los clips se crean enviando una POST solicitud a:

    https://api.bcovlive.io/v1/vods

    Solicitud de cuerpo - Video Cloud

    Nota: se recomienda encarecidamente no especificar el perfil de ingest que se debe utilizar para transcodificar el clip y, en su lugar, utilizar el perfil predeterminado de la cuenta. Esto reducirá la posibilidad de errores en la creación del clip en Video Cloud

    Ejemplo 1: tiempos de inicio/fin relativos al inicio de la secuencia

    El cuerpo de la solicitud incluye las horas de inicio y finalización, así como detalles sobre dónde enviar el clip. Aquí hay un cuerpo de solicitud de muestra que crea un clip del tercer minuto de una secuencia y lo envía a una Video Cloud cuenta:

    {
  "live_job_id":"PUT-LIVE-JOB-ID-HERE",
  "outputs":[
    {
      "label": "60 secs by stream from min 2 to min 3",
      "stream_start_time": 120,
      "stream_end_time": 180,
      "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL",
      "videocloud": {
        "video": {
        "name": "One Minute Clip",
        "tags": ["live", "clip"]
        },
          "ingest": {
            "capture-images": true
        }
      }
    }
  ]
}

    En este ejemplo, estamos creando un clip de un minuto de duración y enviándolo a Video Cloud . Estamos dando al clip un nombre y un par de etiquetas, no especificando el ingest profile para retranscodificación, de modo que se usará la cuenta predeterminada, e indicaremos que Video Cloud capturar imágenes en miniatura y póster del clip durante transcodificación.

          },

      }
    }
  ]
}

    En este ejemplo, estamos creando un clip de un minuto de duración en una época específica (en este caso, 22 de enero de 2018 a las 08:24:54 GMT).

    El cuerpo de la solicitud incluye las horas de inicio y finalización en tiempo de época, y detalles sobre dónde enviar el clip. Aquí hay un cuerpo de solicitud de muestra que crea un clip del tercer minuto de una secuencia y lo envía a una Video Cloud cuenta:

    En este ejemplo, estamos creando un clip de un minuto de duración. Dado que no estamos especificando una hora de inicio o finalización, el clip se tomará de los últimos 60 segundos de la transmisión en vivo.

    Información general sobre el envío de clips a Video Cloud

    Para ver qué campos se pueden incluir en el video y ingest objetos, ver el Dynamic Ingest API Reference.

    No incluya el master campo en el ingest objeto, ya que esa información será proporcionada por la API Live.

    Cuerpo de solicitud - S3

    El cuerpo de la solicitud incluye las horas de inicio y finalización, así como detalles sobre dónde enviar el clip. Aquí hay un cuerpo de solicitud de muestra que crea un clip del tercer minuto de una secuencia y lo envía a un depósito de S3:

    {
  "live_job_id":"",
  "outputs":[
    {
      "label": "last_30",
      "duration": 30,
      "url": "s3://YOUR_BUCKET_NAME/file_name.mp4",
      "credentials": "s3-credentials",
      "notifications": ["http://myserver.com/api/notification_listener?type=jvod"]
    }
  ],
}

    En este ejemplo, estamos creando un clip de 30 segundos de duración y enviándolo a un depósito de S3. Proporcionamos la URL del depósito, incluido el nombre del archivo para el clip, y una cadena que es el nombre de las credenciales de depósito de S3 guardadas. Brightcove Support puede configurar las credenciales para su cuenta.

    Respuesta de API

    La respuesta a una solicitud de creación de clip incluye un identificador para el trabajo y la etiqueta establecida en el cuerpo de la solicitud, así como el id de trabajo activo:

    {
  "vod_jobs": [
    {
