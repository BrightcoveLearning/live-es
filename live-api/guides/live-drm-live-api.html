Live DRM a través de Live API

Este tema cubre cómo agregar DRM para transmisiones en vivo a través de LIve API.

Introducción

DRM se puede agregar a los trabajos en vivo al incluir el DRM objeto en el cuerpo de la solicitud cuando crea el trabajo. La DRM El objeto tiene dos propiedades:

  • modes establecido en una matriz o uno o más de los siguientes: "all", "fairplay", "playready", "widevine", "widevine: dash" o "widevine: hls"
  • token_expires_in establecido en un número entre 43200 (predeterminado) y 31622400 (el número de segundos antes de que expire el token)

Configuración mode to all dará como resultado que todos los tipos de DRM disponibles se apliquen a las salidas. De forma predeterminada, la DRM se aplica a las salidas de la siguiente manera:

  • FairPlay: aplicado a las salidas de HLS
  • Playready: aplicado a las salidas DASH
  • Widevine: aplicado a salidas HLS

Widevine se puede aplicar a DASH salidas en lugar de HLS. Para que Widevine con DASH sea el predeterminado, se requiere una anulación a nivel de cuenta. Comuníquese con Soporte si desea que esta anulación se aplique a su cuenta. En la siguiente sección verá cómo puede especificar widevine:hls o widevine:dash para trabajos particulares, independientemente del valor predeterminado de su cuenta. Tenga en cuenta que, según la configuración de su cuenta / trabajo, la respuesta de la API (como la que se muestra a continuación) incluirá widevine:hls o widevine:dash salidas, pero no ambas.

Tenga en cuenta que existen límites de simultaneidad para PlayReady definidos en su contrato de servicio con Microsoft. Ver este documento de Microsoft para obtener información sobre la gestión de la concurrencia. Si se supera el límite de concurrencia de un vídeo, no se reproducirá para los espectadores más allá del número máximo.

Trabajo con Widevine-HLS

Para crear DASH empaquetado con Widevine salidas (sin obtener una anulación de cuenta para que esto sea el predeterminado), especifique widevine:dash en el modes como se muestra en el ejemplo siguiente. De manera similar, si ha configurado su cuenta por defecto para usar Widevine con DASH, puede obtener salidas Widevine-HLS para un trabajo en particular especificando widevine:HLS en el modes.
