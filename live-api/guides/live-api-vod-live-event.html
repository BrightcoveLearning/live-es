    API en vivo: VOD del Evento en Vivo

    En este tema, aprenderá cómo crear un vídeo bajo demanda (VOD) a partir de un evento de transmisión en vivo una vez finalizado el evento.
    Live to VOD no se puede usar con transmisiones cifradas y devolverá un error al crear un evento de trabajo Live.

    Introducción

    En el caso de eventos de transmisión en directo más cortos, es posible que desee que una grabación del evento esté disponible como vídeo bajo demanda (VOD) en su Video Cloud cuenta una vez finalizado el evento. Puede hacerlo fácilmente agregando algo de configuración para el VOD cuando cree su trabajo en vivo.

    Fast Live to VOD permite que los clips se publiquen rápidamente con precisión de segmentos, mientras que un proceso de back-end totalmente transparente para el cliente limpiará los límites de los clips y volverá a publicar.

    El resultado es un tiempo de comercialización más rápido con un clip que volverá a publicar automáticamente con los límites de clip deseados.

    Importante:

    • Cuando se complete el evento de transmisión en vivo, detenga el codificador, pero no cancele el trabajo. Espere hasta que se haya creado el clip VOD antes de cancelar el trabajo, o no se creará correctamente.
    • La creación del VOD no comenzará hasta que el reconnect_time para el trabajo en vivo haya expirado.
    • Si la duración del evento en directo es superior a 24 horas, solo las últimas 24 horas se capturarán en el VOD.

    Opción VOD rápida

    La opción VOD rápida le permite obtener la versión VOD del evento en vivo más rápidamente. Fast Live to VOD publicará (usando un transmux) de representaciones en vivo que serán segmentos precisos.

    Los output campos para VOD rápido son:

    • mode ( requerido para VOD rápido) - para VOD rápido, el valor será instant
    • playlist_label ( opcional): solo aplicable para VOD en modo instantáneo. Indica qué lista de reproducción se va a utilizar para crear una salida VOD (a partir del trabajo activo). Solo es necesario si se definieron listas de reproducción personalizadas en la creación de trabajos en vivo.

    Implementación

    Enviar a Video Cloud

    Tenga en cuenta que hay dos formas en que videocloud los objetos se utilizan al crear un nuevo trabajo activo:

    1. Incluyendo videocloud en los campos de nivel de trabajo creará un video en la Video Cloud cuenta que tomará la transmisión en vivo como un activo remoto: 
      {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 2400,
    "live_dvr_sliding_window_duration": 30,
    "videocloud": {
  		"video": {
  			"name": "test_live_stream"
  		}
  	}...

      En este caso, no hay ningún ingest objeto dentro del videocloud objeto, porque no se ingerirá nada.

    2. Incluir el videocloud objeto en las propiedades de un outputs clip o de un objeto VOD completo creará un Video Cloud vídeo para el clip; el clip se ingiere en Video Cloud y pasa a formar parte de la biblioteca VOD:

      {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 2400,
    "live_dvr_sliding_window_duration": 30,
	"videocloud": {
		"video": {
			"name": "test_live_stream"
		}
	},
  "outputs": [
      { "videocloud": {
          "video": {
              "name": "video name",
          },
          "ingest": {
              "profile": "valid-ingest-profile-name"
          }
      },
      "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL"
      } 
  ]
}

    Para tener una versión VOD del evento guardada en su Video Cloud cuenta, incluya una versión de la salida resaltada a continuación.

    {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 600,
    "ad_insertion": false,
    "outputs": [
        { OTHER_JOB_OUTPUT_OBJECT },
        { OTHER_JOB_OUTPUT_OBJECT_2 },
        { "videocloud": {
            "video": {
                "name": "video name",
                "description": "video description",
                "long_description": "long description",
                "tags": [
                "tag1", "tag2"
                ],
                "schedule":
                { "starts_at": "2017-01-01T12:00:00-04:00", "ends_at": "2099-01-01T12:00:00-04:00" }
                ,
                "custom_fields": {
                    "someCustomField": "value"
                }
            },
            "ingest": {
                "profile": "valid-ingest-profile-name"
            }
        },
        "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL"
        } 
    ]
}
    Si aún no tiene credenciales para su depósito de S3 configurado, puede agregar nuevas credenciales a través de la API.

    Tenga en cuenta que para los metadatos de vídeo, puede incluir los campos enumerados para la operación Crear vídeo de la API de CMS.

    Los perfiles de ingesta definen las copias que se crearán para el VOD. Para valid-ingest-profile-name, puede sustituir el nombre de cualquier perfil de ingesta personalizado o estándar para su cuenta. Los perfiles estándar que tengas dependen de si tu cuenta está habilitada o no para Entrega dinámica. Consulte Perfiles de ingesta estándar para entrega dinámica y codificación consciente del contexto o Perfiles de ingesta estándar para transcodificación heredada.

    Tenga en cuenta que el profile campo es opcional. Si no especifica un perfil, se utilizará el perfil predeterminado para la cuenta.

    Enviar a S3

    Para tener una versión VOD del evento guardada en su depósito de S3, incluya una versión de la salida resaltada a continuación.

    {
    "live_stream": true,
    "region": "us-west-2",
    "reconnect_time": 600,
    "ad_insertion": false,
    "outputs": [
        { OTHER_JOB_OUTPUT_OBJECT },
        { OTHER_JOB_OUTPUT_OBJECT_2 },
        {
        "url":"s3://YOUR_BUCKET/path/filename.mp4",
        "credentials": "YOUR_CREDENTIAL_LABEL",
        "notifications":    [{
            "url": "http://httpbin.org/post?vodStateChange"
        },
        {
            "url": "http://httpbin.org/post?vodFinished",
            "event": "output_finished"
        }]
    }
    ]
}
    Se debería haber creado una etiqueta de credencial para un nombre de usuario+contraseña válido+para su depósito de S3 cuando se configuró su cuenta Live y se le envió la etiqueta de credencial. Si no lo tiene, póngase en contacto con el servicio de soporte de Brightcove.

    Limitaciones

    • Si detiene (cancela) el trabajo activo antes de que se haya creado el VOD, no se creará. En su lugar, detenga el codificador para iniciar la creación del VOD y espere a que termine antes de detener el trabajo.
    • Para transmisiones de punto de entrada estático solamente , no se puede configurar Live to VOD para archivar la secuencia al finalizar.

      Alternativas

      • Utilice el recorte para archivar el evento en vivo completo como un VOD. La ventana de recorte estará disponible para la duración estándar después de un evento o hasta la próxima activación del SEP (la que sea más corta).
