    API en vivo: Insertar metadatos temporizados ID3

    En este tema se explica cómo insertar metadatos temporizados ID3 en la transmisión en vivo mediante Live API.

    Resumen

    Los metadatos cronometrados ID3 se utilizan para enviar datos con una transmisión en vivo en un punto específico en el tiempo. Para insertar metadatos temporizados ID3 para un trabajo en curso, realice una POST solicitud a:

    https://api.bcovlive.io/v1/jobs/JOB_ID/id3tag

    Cuerpo de solicitud de muestra

    {
 "id3_tag": {
 "name": "cp1",
 "value": "Some information I want to send",
   "timecode": "15:50:49:16"
 }
}
    Nota:
    • Sólo se puede enviar un id3_tag objeto en una solicitud
    • Si utiliza la timecode propiedad, el trabajo sólo almacena la solicitud de inserción más reciente
    • Si utiliza la timecode propiedad, el codificador debe enviar código de tiempo con formato SMPTE (HH:MM:SS:FF) almacenado en la propiedad tc a través de onFi
    • Los codificadores de software como Wirecast y OBS no admiten el envío de código de tiempo a través de paquetes OnFi en la secuencia RTMP
    • Los codificadores de hardware elementales admiten el código de tiempo de envío a través de paquetes OnFi en la secuencia RTMP
    • La etiqueta de estados ID3v2 name puede tener hasta 4 caracteres.
    • value puede ser de hasta 256 KB.
    • Timecode es opcional: cuando se omite, la inserción será inmediata.

    Respuesta de muestra

    {
"id": "JOB_ID",
"id3_tag": {
"tag_name": "BCOV",
"tag_value": "my value"
}
}

    ¿Existen códigos de tiempo SMPTE en la entrada?

    No todos los codificadores envían códigos de tiempo SMPTE. Para verificar si existen en su entrada, una forma de verificar es mirar la lista de chunklist VOD para etiquetas como #VOD-TIMING:timecode=15:18:36:01,wallclock=1549047672441,tsvideo=25400,tsaudio=-1. La URL de lista de chunklist vod debe estar disponible en output_media_files[0].playback_url_vod cuando recupere la información del trabajo.

    Analizar una lista de reproducción en vivo para obtener el código de tiempo

    Cuando crea un trabajo de Brightcove Live, se devuelve una lista de reproducción principal predeterminada. Por ejemplo:

    https://bcovlive-a.akamaihd.net/ade7a7b3aaea4a4d8107fb8bb262381e/eu-central-1/52725330351/playlist.m3u8

    Necesitaremos la lista de reproducción maestra VOD para la que tendremos que añadir «_vod» en la última parte de nuestra extensión URL como playlist_vod.m3u8

    https://bcovlive-a.akamaihd.net/ade7a7b3aaea4a4d8107fb8bb262381e/eu-central-1/52725330351/playlist_vod.m3u8

    La lista de reproducción estará compuesta por varias representaciones/perfiles, que si «curl» la lista de reproducción, podrá ver las copias. Ejemplo:

    
curl -i "https://bcovlive-a.akamaihd.net/ade7a7b3aaea4a4d8107fb8bb262381e/eu-central-1/52725330351/playlist_vod.m3u8"

    https://bcovlive-a.akamaihd.net/ade7a7b3aaea4a4d8107fb8bb262381e/eu-central-1/52725330351/profile_1/chunklist_vod.m3u8

    2. Si utilizas el navegador Safari, puedes reproducir la lista de reproducción maestra o una de las listas de repartidas. A continuación, abra la pestaña de red de desarrolladores y busque «chunklist». Seleccione uno de ellos y mire la pestaña de vista previa.
    3. Si no dispone de Safari, puede aplicar el mismo proceso en otro navegador utilizando el reproductor de Akamai en http://players.akamai.com/hls/. Cargue la lista de reproducción en el reproductor de Akamai y, a continuación, abra la pestaña de red del navegador y busque la «lista de elementos».
    4. Si usted tiene un proxy, como «Charles», es el mismo principio. Reproduce una de las listas de reproducción VOD en el reproductor Safari o Akamai HLS y, a continuación, filtra los resultados en tu proxy para ver una de las listas de trozos devueltas.
