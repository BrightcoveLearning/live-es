    API en vivo: Puntos de entrada estáticos

    En este tema se explican los puntos de entrada estáticos y cómo implementarlos mediante la API Live.

    Introducción

    La función de puntos de entrada estáticos (SEP) permite un trabajo en vivo de larga duración que se puede activar y desactivar mientras mantiene las URL de punto de entrada y las URL de reproducción estáticas y reutilizables. Esta función permite a los clientes configurar su codificador en sus instalaciones o en el campo y permite al cliente crear su propia lógica de programación para canales o programas en vivo.

    DVR y SEP

    La capacidad DVR está disponible para puntos de entrada estáticos, pero solo mientras SEP está activado.

    Creación de un punto de entrada estático

    Se utilizará la operación de creación de trabajo estándar, pero para la adición de puntos de entrada estáticos se necesitará agregar una propiedad a la solicitud de trabajo:

    "static": true

    La URL RTMP que responde de este trabajo será estática y luego se puede reutilizar activando o desactivando el trabajo con las operaciones descritas en las secciones siguientes. Lea y comprenda las limitaciones enumeradas al final de este documento.

    Tiempo máximo de espera

    Después de activar un trabajo SEP, se desactivará automáticamente después de algún tiempo si el codificador no se inicia. Las reglas son las siguientes:

    • Si el event_length valor es mayor que 30 minutos, el trabajo finalizará automáticamente en 30 minutos de forma predeterminada; si desea cambiar el tiempo de espera, deberá ponerse en contacto con el Soporte técnico. No se puede desactivar la desactivación automática.
    • Si el event_length valor es menor que el reconnect_time, el trabajo finalizará en el tiempo establecido para event_length.

    Activar SEP Live Stream

    Para activar una transmisión en vivo de punto de entrada estática, envíe la siguiente solicitud a la API Live:

    Para poder ingerir en un punto de entrada estático, debe realizar la solicitud activate API.
    Secuencia SEP activa
    Método PUT
    Endpoint 
    https://api.bcovlive.io/v1/jobs/{jobID}/activate

    No se requiere ningún organismo de solicitud.

    Desactivar SEP Live Stream

    Detenga siempre el codificador antes de desactivar la secuencia. Es posible que la desactivación no ocurra inmediatamente, por lo que esto le da un control exacto sobre cuándo los espectadores dejan de recibir contenido.

    Para desactivar una transmisión en vivo de punto de entrada estática, envíe la siguiente solicitud a la API Live:

    Secuencia SEP activa
    Método PUT
    Endpoint 
    https://api.bcovlive.io/v1/jobs/{jobID}/deactivate
    Cuando desactiva un punto de entrada estático, los últimos segundos de la transmisión permanecen reproducibles.

    No se requiere ningún organismo de solicitud.

    Cancelar un trabajo SEP

    Para detener permanentemente un trabajo de punto de entrada estático, puede cancelar el trabajo del mismo modo que lo haría con cualquier trabajo activo, enviando una PUT solicitud al punto final /jobs/{job_id}/cancel. Consulte la referencia de API para obtener más detalles.

    Limitaciones

    • Live to VOD no se puede configurar para archivar la transmisión al finalizar.

      Alternativas

      • Utilice el recorte para archivar el evento en vivo completo como un VOD. La ventana de recorte estará disponible para la duración estándar después de un evento o hasta la próxima activación del SEP (la que sea más corta).
    • No se puede modificar la configuración del trabajo. Actualmente están vinculados a la configuración que creó inicialmente el SEP.
    • La activación de la transmisión puede tardar hasta 60 segundos para que el punto de entrada esté disponible.

      Práctica recomendada:

      Implemente notificaciones desde la plataforma Live para los cambios en el estado del trabajo para obtener actualizaciones en tiempo real cuando un trabajo SEP está realmente listo.

      Alternativas

      1. Active la secuencia al menos 60 segundos antes de iniciar el codificador.
      2. Si el codificador tiene reintentos incorporados (Wirecast, Elemental, etc.), el flujo se puede conectar inmediatamente (aunque puede haber algunos errores) y los codificadores se reconectarán tan pronto como el punto de entrada esté disponible.
    • Reconnect_time sigue siendo aplicable para trabajos SEP en desconexiones del codificador. Si se alcanza el tiempo de reconexión, el trabajo SEP volverá a estar desactivado.

      Práctica recomendada:

      Desactive siempre una secuencia a través de las API cuando finalice un trabajo y establezca el tiempo de reconexión a una ventana aceptable de tiempo para esperar en caso de desconexión (los costes de la hora de entrada se acumulan durante este período de espera).

    • A cada cuenta solo se le asignarán 3 puestos de trabajo SEP disponibles por región. Si alcanza el límite, deberá cancelar uno de sus trabajos de SEP antes de poder crear uno nuevo.

      Si ha alcanzado el número máximo de trabajos SEP e intenta crear otro, se devolverá el siguiente error:

      {
  "error": "You've reached the number of static entry points jobs you can create for your account"
}

    Notas

    Recorte de trabajos SEP después de la desactivación
    Los usuarios pueden crear clips hasta la próxima activación de la transmisión, o durante 7 días, lo que sea más corto. Cuando se vuelve a activar la transmisión, las sesiones de grabación anteriores se eliminan del servidor y ya no se pueden recortar.
    ¿Cuál es el tiempo para el campo stream_start_time cuando se utiliza un SEP?
    Cuando se activa un SEP, se consideraría la hora de inicio.
