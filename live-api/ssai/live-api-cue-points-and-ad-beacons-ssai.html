    API en vivo: Puntos de referencia y balizas de anuncio con SSAI

    En este tema, aprenderá a administrar los puntos de referencia y los indicadores de anuncio cuando utilice la Inserción de anuncios del lado del servidor (SSAI) de Brightcove para sus trabajos de transmisión en directo.

    Resumen

    La inserción de anuncios en el lado del servidor (SSAI) le permite mostrar anuncios durante un evento de transmisión en vivo en momentos especificados. Para obtener información general, consulte la API Live: Inserción de anuncios en el servidor (SSAI) documento.

    Antes de poder utilizarlo SSAI, debe estar habilitado para su cuenta. Consulte los detalles al gestor de su cuenta.

    Puntos de referencia

    Los saltos de anuncios se activan mediante puntos de referencia, que se pueden especificar de dos maneras:

    • Enviado a Brightcove por el codificador
    • Puntos de referencia inmediatos creados a través del Live API

    Desde el codificador

    El sistema de entrega en directo de Brightcove puede interpretar los puntos de referencia enviados por el codificador en el formato AMF:

      AMFDataList
  [0]:onCuePoint
  [1]:{Obj[]:
    time: 1.9889, //Difference from PTS of THIS packet to the first PTS of the 1st video frame in the adbreak
    name: "scte35",
    type: "event",
    ad_server_data: "YWJjZGVmZ2g=",	// optional introduced by Brightcove. It is a base64 encoded json map of parameters e.g. {‘key’:’value’}
    parameters: {Obj[]:
      type: "avail_in",
      duration: 12.0
    }
  }

    Notas:

    • Actualmente sólo se admiten los puntos de referencia de avail_in tipo.
    • Los puntos de referencia SCTE-35 son compatibles con las entradas RTMP y TS .
    Consulte el apéndice a continuación para obtener una captura de pantalla de una configuración de punto de referencia de muestra para el codificador Elemental.

    Inserción manual de puntos de referencia

    Puede crear puntos de referencia inmediatos Live API mediante el envío de una POST solicitud:

    Método POST
    URL https://api.bcovlive.io/v1/jobs/Job_ID/cuepoint
    Encabezado X-API-KEY: your API KEY

    Incluya un cuerpo de solicitud que especifique lo siguiente:

    Campo Tipo Descripción
    duration Entero Duración del descanso en segundos.

    La duración del punto de referencia que se inserta debe ser al menos el doble de la longitud de los segmentos del trabajo. Consulte el ejemplo de duración.
    timecode Formato SMPTE OPCIONAL: Un código de tiempo en formato SMPTE, HH:MM:SS:FF (FF = frames), para especificar cuándo se debe pasar un conjunto de variables (pares clave/valor) al AdServer.

    Si se omite, el punto de referencia se insertará inmediatamente.

    Si utiliza la propiedad timecode, el codificador debe enviar código de tiempo con formato SMPTE (HH:MM:SS:FF) almacenado en la tc propiedad a través de OnFI. Los códigos de tiempo son desde el inicio de la transmisión en vivo.
    ad_server_data Objeto OPCIONAL: Los pares clave/valor que pase dependerán del servidor de anuncios que esté utilizando. Para obtener más información, consulte la documentación del servidor de anuncios y la sección Segmentación de anuncios mediante macros de anuncios.

    Ejemplo de duración

    La duración del punto de referencia que se inserta debe ser al menos el doble de la longitud de los segmentos del trabajo.

    Por ejemplo, insertar un punto de referencia de 10 segundos en un trabajo con "segment_seconds"=4 , funcionará bien. Sin embargo, insertar el mismo punto de referencia en un trabajo con "segment_seconds"=6 dará como resultado el siguiente error:

      "error": "The parameter duration should be greater than
    or equal to (2 * target duration) of the job"
     

    Cuerpo de solicitud de muestra

      {
    "duration": 30,
    "timecode": "15:50:49:16",
    "ad_server_data" : {
    "adbreakid": 12312
    "breaktheme": "fitness"
    }
  }

    Notas

    1. Los codificadores de software como Wirecast y OBS no admiten el envío de código de tiempo a través de OnFI paquetes en la secuencia RTMP
    2. Los codificadores de hardware elementales admiten el código de tiempo de envío a través de OnFI paquetes en la secuencia RTMP

    Respuesta de muestra

      {
    "id": "Job_ID",
    "cue_point": {
      "id": "adBreak-2f58393ada1442d98eca0817fa565ba4",
      "duration": 30,
      "accuracy": "segment", [Can be segment or frame  ]
      "inserted_at": "2017-07-21T09:30:46.307Z" [ Time when the cue point was inserted in the stream]
    },
  }

    Balizas

    Los indicadores son puntos de datos de reproducción enviados a análisis de terceros para realizar un seguimiento de si se han reproducido anuncios y cuántos de ellos se han reproducido. En esta sección vamos a ver los tipos de balizas que se pueden establecer usando las variables Live API, y que se pueden usar para proporcionar los datos. En la siguiente sección se detallará el uso de las solicitudes de API para crear y administrar conjuntos de balizas.

    Tipos de balizas

    Tipos de balizas
    Tipo de baliza Descripción
    Load Se dispara una vez por sesión y solo se activa cuando se solicita el manifiesto de nivel superior
    Play Se ha solicitado contenido y se ha devuelto el primer segmento
    Heartbeat Duración objetivo (segmentos segundos)
    AdStart Anuncio individual iniciado
    AdFirstQuartile Primer cuartil de anuncios (25%)
    AdMidpoint Segundo cuartil de anuncios (50%)
    AdThirdQuartile Tercer cuartil de anuncios (75%)
    AdComplete Anuncio individual completado
    AdBreakStart Se ha iniciado la interrupción de anuncios
    AdBreakComplete ha finalizado la interrupción del anuncio

    Variables de baliza/anuncio

    La siguiente tabla muestra las variables que puede utilizar para proporcionar datos para las URL de indicadores. Para incluir una variable, rodea con llaves dobles, como esta: {{job.job_id}} . Consulte la siguiente sección sobre administración de conjuntos de balizas para ver ejemplos completos.

    Variables de configuración de anuncios
    Variable
    Descripción
    session.session_id
    ID de sesión única
    job.job_id
    ID de trabajo único
    videocloud.video_id
    Solo disponible para trabajos creados con un vídeo VideoCloud.
    application_ad_configuration.description
    valor de la aplicación en la creación de la sesión
    random.int32
    Entero aleatorio de 32 bits con signo
    random.int64
    Entero con signo aleatorio de 64 bits
    random.uint32
    Entero sin signo aleatorio de 32 bits
    random.uint64
    Entero aleatorio de 64 bits sin signo
    random.uuid
    uuid aleatorio
    server.timestamputc
    tiempo de época en milisegundos cuando se ha realizado la llamada desde el ads-api
    client.useragent
    valor de encabezado http user-agent en la creación de la sesión
    client.ipaddress
    http x-forwarded-for valor de encabezado en la creación de la sesión, si se proporciona, de lo contrario, la dirección remota
    client.referrer
    valor de encabezado de referencia http en la creación de la sesión (nota: esa es la ortografía correcta)
    client.referer
    valor de encabezado de referencia http en la creación de la sesión (ortografía http)
    client.ipuaid
    valor hash de client.ipaddress y client.useragent
    live.adbreak
    (actualmente no utilizado)
    live.adbreakdurationms
    Duración de interrupción del anuncio en milisegundos
    live.adbreakduration
    Duración de la interrupción del anuncio en segundos de coma flotante de precisión doble
    live.adbreakdurationint
    Duración de interrupción del anuncio en segundos enteros
    live.adbreak.timestamputc.wallclock
    tiempo de época en milisegundos cuando se ha realizado la llamada al servidor de anuncios
    live.adbreak.timestamputc.origin
    tiempo de época en milisegundos desde la lista de elementos de origen. Este valor indica la hora en que se ha creado el fragmento de salida en la lista de fragmentos de origen.
    live.adbreak.timestamputc.session
    tiempo de época en milisegundos de la lista de chunklist ssai. Este valor indica el tiempo del fragmento de salida de salida en la lista de fragmentos ssai. Dado que el contenido de adbreak y la brecha de adbreak NO suelen ser los mismos, después del primer adbreak el live.adbreak.timestamputc.origin es diferente de live.adbreak.timestamputc.session . Este valor tiene en cuenta los ajustes de tiempo que se han realizado en la lista de SSAI chunklist.

    Variables de anuncio SCTE-35

    La Sociedad de Ingenieros de Telecomunicaciones por Cable (SCTE) define un estándar para la inserción dinámica de anuncios para transmisiones en directo. En la tabla siguiente se resumen las variables de configuración de anuncios SCTE-35.

    Los parámetros de anuncio SCTE-35 se admiten actualmente con fuentes de entrada RTMP y Transport Stream (TS) utilizando el Protocolo de transporte en tiempo real (RTP) con o sin corrección de errores de envío (FEC). Tenga en cuenta que FEC es muy recomendable para RTP.

    Estos parámetros de anuncio no están disponibles con las secuencias de entrada que utilizan el Protocolo de mensajería en tiempo real (RTMP).
    Variables de configuración de anuncios SCTE-35
    Variable
    Descripción
    scte35_eventID
    un identificador de evento único pasado con un mensaje SCTE35.
    scte35_programID
    Un identificador de programa único pasado con un mensaje SCTE35.
    scte35_availNum
    Un identificador de un tiempo de empalme específico disponible para los anuncios, se envía a través de un mensaje SCTE35.
    scte35_breakDuration
    Duración de la interrupción del anuncio, en términos de ticks del reloj de 90 kHz del programa, transmitida con un mensaje SCTE35.
    scte35_spliceTime
    Tiempo de empalme para un descanso publicitario, en términos de ticks del reloj de 90 kHz del programa, pasado con un mensaje SCTE35.

    Administración de conjuntos de balizas

    Esta sección proporciona detalles sobre las solicitudes de API para administrar conjuntos de balizas. Consulte la sección anterior para ver los tipos y variables de balizas.

    Para agregar un conjunto de balizas a un trabajo activo, primero cree el conjunto de balizas y, a continuación, incluya el identificador al crear el trabajo, de esta manera:

      {
  "live_stream": true,
  "region": "us-west-2",
  "reconnect_time": 30,
  "ad_insertion": true,
  "beacon_set": "beacon_set_id", ...

    Crear un conjunto de balizas

    Para crear un conjunto de balizas, envíe una POST solicitud:

    Método POST
    URL https://api.bcovlive.io/v1/ssai/beaconsets
    Encabezado X-API-KEY: your API KEY

    Cuerpo de solicitud de muestra

      {
    "account_id": "User's Account ID [Optional]",
    "beacon_urls": [
      {
        "beacon_url": "https://myserver.com/beaconRX/{{job.job_id}}/load?position=load&sid={{session.session_id}}&jid={{job.job_id}}&app={{application_ad_configuration.description}}&rnd32={{random.int32}}&rnd64={{random.int64}}&bid={{random.uuid}}&t={{server.timestamputc}}&ua={{client.useragent}}&ip={{client.ipaddress}}&ref={{client.referrer}}&ref={{client.referer}}&ab={{live.adbreak}}&abd={{live.adbreakduration}}&abdi={{live.adbreakdurationint}}",
        "beacon_type": "Load"
      },
      {
        "beacon_url": "https://myserver.com/beaconRX/{{job.job_id}}/play?position=play&sid={{session.session_id}}&jid={{job.job_id}}&app={{application_ad_configuration.description}}&rnd32={{random.int32}}&rnd64={{random.int64}}&bid={{random.uuid}}&t={{server.timestamputc}}&ua={{client.useragent}}&ip={{client.ipaddress}}&ref={{client.referrer}}&ref={{client.referer}}&ab={{live.adbreak}}&abd={{live.adbreakduration}}&abdi={{live.adbreakdurationint}}",
        "beacon_type": "Play"
      }
    ]
  }
    El campo es el ID de cuenta activa.account_id Cuando se omite, se utiliza el ID de cuenta del usuario solicitante.

    Respuesta de muestra

      {
    "beacon_set": {
      "beacon_urls": [{
      "beacon_url": "https://myserver.com/beaconRX/{{job.job_id}}/load?position=load&sid={{session.session_id}}&jid={{job.job_id}}&app={{application_ad_configuration.description}}&rnd32={{random.int32}}&rnd64={{random.int64}}&bid={{random.uuid}}&t={{server.timestamputc}}&ua={{client.useragent}}&ip={{client.ipaddress}}&ref={{client.referrer}}&ref={{client.referer}}&ab={{live.adbreak}}&abd={{live.adbreakduration}}&abdi={{live.adbreakdurationint}}",
      "beacon_type": "Load"
    },
    {
      "beacon_url": "https://myserver.com/beaconRX/{{job.job_id}}/play?position=play&sid={{session.session_id}}&jid={{job.job_id}}&app={{application_ad_configuration.description}}&rnd32={{random.int32}}&rnd64={{random.int64}}&bid={{random.uuid}}&t={{server.timestamputc}}&ua={{client.useragent}}&ip={{client.ipaddress}}&ref={{client.referrer}}&ref={{client.referer}}&ab={{live.adbreak}}&abd={{live.adbreakduration}}&abdi={{live.adbreakdurationint}}",
      "beacon_type": "Play"
    }],
    "beacon_set_id": "Inserted Beacon Set ID",
    "account_id": "USER's ACCOUNT ID"
    }
    "inserted": true
  }

    Actualizar un conjunto de balizas

    La actualización de un conjunto de balizas es similar a la creación de uno. Enviar una PUT solicitud:

    Método PUT
    URL https://api.bcovlive.io/v1/ssai/beaconsets/beaconset/beacon_set_id
    Encabezado X-API-KEY: your API KEY

    Cuerpo de solicitud de muestra

      {
    "account_id": "User's Account ID [Optional]",
    "beacon_urls": [
      {
      "beacon_url": "https://myserver.com/beaconRX/load",
      "beacon_type": "Load"
      },
      {
      "beacon_url": "https://myserver.com/beaconRX/play",
      "beacon_type": "Play"
      }
    ]
  }
    El campo es el ID de cuenta activa.account_id Cuando se omite, se utiliza el ID de cuenta del usuario solicitante.

    Respuesta de muestra

      {
    "beacon_set": {
      "account_id": "User's Account ID",
      "beacon_set_id": "Beacon set ID",
      "beacon_urls": [{
        "beacon_url": "https://myserver.com/beaconRX/load",
        "beacon_type": "Load"
        },
        {
        "beacon_url": "https://myserver.com/beaconRX/play",
        "beacon_type": "Play"
      }],
      "updated_beacon_set": {
        "beacon_set_id": "Beacon set ID",
        "beacon_urls": [{
          "beacon_url": "https://myserver.com/beaconRX/load",
          "beacon_type": "Load"
        },
        {
          "beacon_url": "https://myserver.com/beaconRX/play",
          "beacon_type": "Play"
        }],
        "account_id": "User's Account ID"
      }
    }
  }

    Obtener conjuntos de balizas

    Para recuperar los conjuntos de balizas de una cuenta, envíe una GET solicitud:

    Método GET
    URL https://api.bcovlive.io/v1/ssai/beaconsets/account/Account ID
    Encabezado X-API-KEY: your API KEY
    Tenga en cuenta que el ID de cuenta aquí es el ID de cuenta Live.

    Respuesta de muestra

      [{
      "account_id": "User's Account ID",
      "beacon_set_id": "Beacon set ID1",
      "beacon_urls": [{
      "beacon_url": "https://myserver.com/beaconRX/load",
      "beacon_type": "Load"
    }]
    },
    {
      "account_id": "User's Account ID",
      "beacon_set_id": "Beacon set ID2",
      "beacon_urls": [{
      "beacon_url": "https://myserver.com/beaconRX2/load",
      "beacon_type": "Load"
      },
      {
      "beacon_url": "https://myserver.com/beaconRX2/play",
      "beacon_type": "Play"
      }]
  }]

    Obtener conjuntos de balizas para el usuario solicitante

    También puede obtener los conjuntos de balizas para la cuenta del usuario solicitante sin incluir el ID de cuenta en la URL de solicitud:

    Método GET
    URL https://api.bcovlive.io/v1/ssai/beaconsets
    Encabezado X-API-KEY: your API KEY

    Respuesta de muestra

      [{
      "account_id": "User's Account ID",
      "beacon_set_id": "Beacon set ID1",
      "beacon_urls": [{
      "beacon_url": "https://myserver.com/beaconRX/load",
      "beacon_type": "Load"
    }]
    },
    {
      "account_id": "User's Account ID",
      "beacon_set_id": "Beacon set ID2",
      "beacon_urls": [{
      "beacon_url": "https://myserver.com/beaconRX2/load",
      "beacon_type": "Load"
      },
      {
      "beacon_url": "https://myserver.com/beaconRX2/play",
      "beacon_type": "Play"
      }]
  }]

    Obtener un beacon establecido por id

    Para recuperar una única baliza establecida por su ID, envíe una GET solicitud:

    Método GET
    URL https://api.bcovlive.io/v1/ssai/beaconsets/beaconset/beacon_set_id
    Encabezado X-API-KEY: your API KEY

    Respuesta de muestra

      {
      "account_id": "User account ID",
      "beacon_set_id": "Beacon set ID",
      "beacon_urls": [{
        "beacon_type": "Load",
        "beacon_url": "https://myserver.com/beaconRX2/load"
    },
    {
      "beacon_type": "Play",
      "beacon_url": "https://myserver.com/beaconRX2/play"
    }]
  }

    Eliminar un conjunto de balizas

    Finalmente, para eliminar un conjunto de balizas, envíe una DELETE solicitud:

    Método DELETE
    URL https://api.bcovlive.io/v1/ssai/beaconsets/beaconset/beacon_set_id
    Encabezado X-API-KEY: your API KEY

    Respuesta de muestra

    La respuesta se verá así:

      {
    "beacon_set_id": "Beacon set ID",
    "deleted": true
  }
