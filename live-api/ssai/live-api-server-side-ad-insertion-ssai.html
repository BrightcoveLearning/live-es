    API Live de Brightcove: Inserción de anuncios en el lado del servidor (SSAI)

    En este tema se muestra cómo configurar la inserción de anuncios del lado del servidor (SSAI) para los trabajos de transmisión en directo.

    Resumen

    SSAI le permite mostrar anuncios durante un evento de transmisión en vivo en momentos especificados. Las características principales incluyen:

    • Inserta anuncios utilizando puntos de referencia enviados desde tu codificador o crea un punto de referencia instantáneo mediante el Live API
    • Ingiere activos de «pizarra» para llenar cualquier tiempo de anuncio no utilizado
    • Evite los bloqueadores de anuncios con anuncios que se insertan en la transmisión en vivo en el lado del servidor
    Antes de poder utilizarlo SSAI, debe estar habilitado para su cuenta. Consulte los detalles al gestor de su cuenta.

    Para crear una transmisión en directo con anuncios del lado del servidor, siga estos pasos:

    1. Revisar la información general acerca de Live API
    2. Cree una configuración de anuncios en vivo. También puede hacerlo en Video Cloud Studio. Consulte las secciones siguientes para obtener información detallada sobre la administración de las configuraciones de anuncios mediante la API en vivo.
    3. Crear recursos de pizarra. Si prefiere una interfaz de usuario, puede administrar pizarras en Studio.
    4. Opcional: Insertar puntos de referencia y balizas de anuncio.
    5. Ahora, ya está listo para crear una transmisión en vivo usando Live API. Si prefiere usar Studio, puede implementar anuncios del lado del servidor en el módulo Live

    Consulte el resto de este documento para obtener información detallada sobre los encabezados personalizados, las variables de configuración de anuncios, el uso de DFP y macros de anuncios.

    Información general

    La siguiente información se refiere a todas las solicitudes de Live API con SSAI.

    URL base

    La URL base para la API en vivo con SSAI es:

        https://api.bcovlive.io/v1/ssai

    Autenticación

    La autenticación para solicitudes requiere un encabezado con una clave API:

        X-API-KEY: your API KEY

    Póngase en contacto con su administrador de cuenta para obtener una clave API asociada a su cuenta.

    Gestión de pizarras

    Las pizarras son sus propios recursos que se utilizan para llenar el tiempo de anuncio no utilizado. Puede usar pizarras para proporcionar un mensaje de «volver a la derecha» o cualquier contenido que le guste.

    A continuación se detallan las solicitudes de API para agregar y administrar recursos de pizarra.

    Añadir activo de pizarra

    Para ingerir un nuevo recurso de origen de pizarra, envíe una POST solicitud:

    Método POST
    URL https://api.bcovlive.io/v1/ssai/slates
    Encabezado X-API-KEY: your API KEY

    Cuerpo de solicitud de muestra

        {
      "source_url": "https://somesourceasset.com/slate-to-ingest.mp4",
      "account_id": "Account_ID [Optional]",
      "source_description": "User identifiable description for the slate  [Optional]"
    }
    account_id Aquí está el ID de cuenta en vivo. Cuando se omite, se utiliza el ID de cuenta del usuario solicitante.

    Respuesta de muestra

        {
      "media_source_asset_id": "New_UUID",
      "account_id": "Account_ID",
      "media_source_asset_default": false,
      "media_source_asset_type": "slate",

      "media_source_asset_status": "transcoding"

    Notas sobre DFP

    Si está obteniendo anuncios de DFP, aquí hay algunas cosas que debe tener en cuenta para ayudar a evitar problemas.

    Etiqueta de anuncio

    Al crear una etiqueta de anuncio para Live, asegúrese de seguir las directrices adecuadas e incluso /live . Consulte el documento de DFP Crear una etiqueta de vídeo maestra manualmente para obtener más detalles.

    Simultánea

    Si espera una gran cantidad de concurrencia, le recomendamos que hable con su equipo de cuenta de DFP.

    Respuestas de anuncios individuales o múltiples

    los singleadresponse y multiadresponse los parámetros no son utilizados actualmente por SSAI.

    Live SSAI solo realiza una única llamada al servidor de anuncios y espera que la respuesta contenga todos los anuncios de la pausa, con la excepción de que seguirá cualquier envoltorio de anuncios necesario con un límite de 5 redirecciones por anuncio. Se aceptan los siguientes formatos de respuesta de anuncios y se analizarán de la siguiente manera:

    • VAST - Respuesta única o un pod de anuncios en una sola respuesta
    • Reglas publicitarias de DFP: agrega todos los anuncios disponibles en la respuesta, incluidos los anuncios definidos antes, medios y postroll
    • XML inteligente: agrega todos los anuncios disponibles en la respuesta, incluidos los anuncios definidos antes, medio y post-roll

    Encabezados personalizados para solicitudes de anuncios

    La SSAI plataforma puede pasar encabezados personalizados con las llamadas a anuncios y todas las balizas utilizadas por la plataforma back-end. Algunos servidores de anuncios, como VideoPlaza, requieren encabezados personalizados.

    Los encabezados personalizados se especifican como un conjunto de pares clave-valor en un ad_configuration_headers objeto, que forma parte de application_ad_configuration (consulte la sección Crear una configuración de anuncio ).

    Notas

    • Los encabezados estándar se manejan de forma predeterminada, como por ejemplo:
      • X-Forwarded-For
      • X-Device-User-Agent
    • Los valores de encabezado pueden utilizar las variables de configuración del anuncio
    • Los valores de encabezado también pueden ser cadenas estáticas
    Los encabezados se pasarán a todas las solicitudes descendentes, incluidos los envoltorios de anuncios o las impresión/cuartiles, por lo que debe usarlos con precaución.

    Segmentación de anuncios mediante macros de anuncios

    Al crear una configuración de anuncio, la etiqueta de anuncio suele tener un aspecto similar a este:

          https://ad-provider-host.com/path/to/ad-handler?ip={{client.ipaddress}}&
      num={{random.int32}}&ses={{session.session_id}}&IDFA={{deviceid}}&
      sitesection={{sitesection}}&breakid={{breakid}}&breaktheme={{breaktheme}}

    Los elementos dentro de las llaves dobles son variables, también denominadas macros de anuncios, que Brightcove Live reemplazará por valores, si existen, cuando envíe la solicitud de anuncio.

    Las variables que serán útiles dependen del servidor de anuncios.

    Los valores de macro de anuncios se pueden proporcionar de las siguientes maneras:

    • Uso de información de encabezado
    • La anexación de la URL
    • Agregar direcciones URL dinámicamente
    • Uso de puntos de referencia manuales

    Uso de información de encabezado

    La información del encabezado, que se detalla en la sección de variables de configuración del anuncio anterior, está disponible para cualquier solicitud. Simplemente especifique las variables que desee con los nombres clave apropiados en la configuración del anuncio.

    La anexación de la URL

    Se pueden agregar valores de sesión adicionales a la URL de la transmisión en vivo, de esta manera:

          http://playback.bcovlive.io/e058d9f2737e18/us-west-2/NA/
      playlist.m3u8?deviceid=123456&sitesection=services"

    Agregar direcciones URL dinámicamente

    Puede agregar direcciones URL dinámicamente, utilizando Javascript y la API de Brightcove Player:

          <!DOCTYPE html>
      <body>
        <video
        id="myPlayerID"
        data-video-id="5975635167001"
        data-account="3737230870001"
        data-player="tIG7lVKBm"
        data-embed="default"
        data-application-id
        class="video-js"
        controls
        width="640"
        height="360"></video>
        <script src="//players.brightcove.net/3737230870001/tIG7lVKBm_default/index.min.js"></script>

        <script type="text/javascript">
        var player = videojs("myPlayerID");

        player.one("loadstart", function(){
          var sourceUrl = player.currentSource();
          console.log(sourceUrl);
          var liveUrlWithParams = sourceUrl.src + "?player_width={width}&player_height={height}&deviceid={deviceid}";

          player.src([{
            "type": "application/vnd.apple.mpegurl",
            "src": liveUrlWithParams
          }]);
        });
      </script>

      </body>

    Tenga en cuenta que los nombres de clave de este ejemplo corresponden a los nombres de las variables de la etiqueta de anuncio mostrada anteriormente.

    Variables de configuración de anuncios

    Las variables de configuración de anuncios, también conocidas como balizas, se pueden utilizar en las solicitudes para administrar configuraciones de anuncios. Para obtener más información, consulte la API Live: Puntos de referencia y balizas de anuncio con documento SSAI .

    Uso de puntos de referencia manuales

    Los valores para saltos de anuncios específicos se pueden enviar con la solicitud de inserción manual de puntos de referencia. Para obtener más información, consulte la API Live: Documento de puntos de referencia y balizas publicitarias .

    Problemas conocidos

    • SSAI requiere que la transmisión de vídeo en vivo tenga una pista de audio.
    • Si el VAST devuelto tiene el mismo ID de anuncio, el servidor no solicitará el contenido del anuncio, aunque la URL del anuncio utilice variables dinámicas para convertirlo en una URL única. Esto no se aplica si utiliza DFP para anuncios.
    • Con DFP, aunque puede utilizar el mismo identificador de anuncio, todavía tiene que haber un ID de creatividad diferente, es decir, no puede realizar un simple intercambio de MediaFile.

    • Si un salto de anuncio tiene una duración inferior a la duración MÁXIMA de la URL del anuncio (min_ad_duration=0&max_ad_duration=30000), si el anuncio se devuelve más tiempo que el salto de anuncio, el anuncio no se reproducirá.

    • No se admiten anuncios VPAID o en los que se puede hacer clic Brightcove Live SSAI.

    • Cuando un salto de anuncio tiene tiempo restante más corto que los segundos de segmento de la transmisión y se muestra una pizarra, la pizarra se muestra durante su duración del segmento y el salto real del anuncio será más largo de lo esperado.

    • La primera vez que el anuncio sea visto por nuestro sistema, no se reproducirá hasta que esté transcodificado y listo para su entrega.
    • Actualmente no se admite VMAP para SSAI en vivo.
