    Inicio rápido: Crear una transmisión en directo mediante la API Live de Brightcove

    Este tutorial le guiará a través de la creación de un trabajo de transmisión en vivo utilizando la API Live.

    Lo que necesitas

    Para completar este tutorial, necesita lo siguiente.

    • Una cuenta configurada para Live API
    • Una clave de API para la API en vivo (que se le proporciona cuando se configura la cuenta)
    • Una cuenta de Video Cloud (no es necesaria para crear el trabajo de transmisión en directo, pero necesaria para configurar la reproducción en Brightcove Player)
    • Si va CMS API a utilizar para crear el vídeo de Video Cloud en lugar de Studio, necesitará credenciales de cliente con permisos de vídeo/lectura y vídeo/escritura; si no está familiarizado con el proceso de obtención de credenciales de cliente, consulte la siguiente sección de autenticación .
    • La capacidad de realizar solicitudes de API REST ya sea a través de cURL o un cliente REST como Insomnia o Postman.

      Para ayudarte, hemos preparado algunas importaciones de Insomnio y Cartero que puedes usar para el ejercicio (en Brightcove Learning Services, preferimos y usamos Insomnio, pero Postman también funcionará si eso es lo que tienes):

      En cada uno de los archivos zip, encontrará una página léame que le ayudará a lo largo de todo.

      • Importación de Insomnia Workspace
      • Importación de colección y entorno de cartero
    • Para crear una transmisión en vivo real, necesitará un codificador de software o hardware para la transmisión en vivo. Las instrucciones a continuación incluirán los pasos para configurar la transmisión en directo mediante Wirecast ( regístrese para una prueba). Si está utilizando un codificador diferente, el proceso será diferente, pero debe ser similar en términos de la información que necesita de la respuesta Live API.

    La forma más fácil de obtener credenciales en la mayoría de los casos es a través de la sección Autenticación de API de administrador de Studio (requiere permisos de administrador en su cuenta). Consulte Administración de credenciales de autenticación de API para obtener más información. En este caso, los permisos que necesita son para compartir relaciones ; necesita permisos de lectura y escritura:

Cualquiera que sea la opción que elija, deberá solicitar los permisos de operación correctos. Lo siguiente se puede usar con cURL o Postman para obtener los permisos adecuados:

    Configurar vídeo de Video Cloud

    Ahora que tenemos la información de la transmisión en directo, configuraremos un vídeo remoto en Video Cloud para que podamos reproducirlo en un reproductor de Brightcove.

    En estos pasos, configuramos el video usando la API de CMS, pero esto también se puede hacer en Video Cloud Studio, si lo prefiere, consulte los pasos alternativos a continuación.

    Si está utilizando Insomnia o Postman, deberá configurar la autenticación OAuth2 utilizando sus credenciales de cliente. Consulte una de las siguientes guías si necesita ayuda:

    • Usar Insomnio para Solicitudes de API
    • Usar Postman para solicitudes de API

    1. En primer lugar, deberá crear un nuevo video haciendo una POST solicitud a:

      https://cms.api.brightcove.com/v1/accounts/account_id/videos

    2. El cuerpo de la solicitud puede incluir varias piezas de metadatos (consulte la referencia de la API para obtener más detalles), pero lo único requerido es name:

      {
    "name": "some name for your live stream"
}

    3. La respuesta contendrá todos los metadatos de vídeo, pero el campo importante es el id, que necesitará para agregar el activo remoto.

    4. Para agregar el flujo HLS, hará otra POST solicitud a:

      https://cms.api.brightcove.com/v1/accounts/account_id/videos/video_id/assets/hls_manifest

    5. Esta vez, el cuerpo de solicitud será:

      {
    "remote_url": "playback_url for your live stream"
}

    Agregar vídeo a un reproductor

    Ahora crearemos una página HTML simple y le agregaremos un reproductor con el video que acabamos de crear.

    Aquí de nuevo utilizaremos Studio para obtener el código de publicación del reproductor; también puede hacerlo usando la API de administración de Player si lo prefiere.
    1. Cree una página HTML simple para probar el reproductor de transmisión en vivo. Puedes incluir cualquier otro contenido que te guste, o ninguno en absoluto.
    2. En Studio Media Module, seleccione el vídeo creado en los pasos anteriores y haga clic en Publicar e incrustar > Web Player:
    3. En el cuadro de diálogo Publicar, seleccione un reproductor (el Reproductor predeterminado de Brightcove está bien):
    4. Desplácese hacia abajo, configure el tamaño del reproductor como desee, y luego haga clic en el bloque de código para seleccionar el código del reproductor estándar: copie este código y péguelo en el cuerpo de su página html:
    5. Guarde la página html en un servidor web remoto o local para que pueda examinarla.
    6. Abra la página en su navegador web. Como no hemos empezado a cocinar al vapor, verá un error multimedia:

    Usar Studio para crear el vídeo

    Estos son los pasos alternativos para crear el video en Studio.

    1. Inicie sesión en Video Cloud Studio.
    2. Vaya al módulo de medios.
    3. En las opciones de la izquierda, haz clic en Agregar vídeo remoto:
    4. En el cuadro de diálogo Agregar vídeo remoto , agregue un nombre de vídeo y copie y pegue el valor playback_url de la respuesta de Live API en el campo URL. A continuación, haga clic en Agregar copia y Guardar:
      Nota: si desea transmitir a través de HTTPS para una transmisión segura, simplemente cambie http en playback_url a https.

    Usar Wirecast para transmitir un evento

    En este apartado configuraremos y utilizaremos Telestream Wirecast para poder retransmitir secuencias en directo. El software Telestream Wirecast es una aplicación para equipos de escritorio que captura la entrada de la cámara y genera una secuencia que puede subirse a una CDN. Tenga en cuenta que existen otras soluciones de codificación tanto de hardware como de software que pueden resultarle más adecuadas para retransmitir secuencias en directo. Puede descargar una versión de prueba del software Wirecast en el sitio web de Telestream.

    Nota: Antes de proceder, deberá tener instalado el software Wirecast y una cámara conectada a su ordenador.

    Para configurar Wirecast para eventos en directo, siga estos pasos.

    1. Abra Wirecast.
    2. Pase el cursor sobre el +, haga clic en Capturar dispositivos y, a continuación, seleccione la cámara.
    3. Haga clic en el botón de flecha ( ) para hacer que la cámara tomara la toma en vivo.
    4. Haga clic en el botón Stream ( ). Se le pedirá que configure los ajustes de salida.
    5. Ajuste Destino a Servidor RTMP. Haga clic en Aceptar.
    6. Para la dirección, utilice el stream_url devuelto desde la API Live (paso 3).
    7. Para el Stream, usa vivo.
    8. ( Opcional) Para crear flujos de salida adicionales, haga clic en Agregar... Al crear secuencias de salida adicionales, quizá desee crear su propio perfil de codificación personalizado para controlar la tasa de bits de salida. Asegúrese de que el nombre de la Secuencia refleja la nueva tasa de bits.
    9. Haga clic en Aceptar para guardar los ajustes de salida.
    10. Haga clic en el botón Stream ( ). La retransmisión en directo debería comenzar.

    Conclusión

    Ya está listo para crear una sesión de transmisión en vivo. Debe agregar lo siguiente a la configuración del codificador:

    • stream_url: agregue el stream_url de su trabajo (la rtmp dirección)
    • stream_name: el nombre de la secuencia debe ser alive

    Una vez iniciada la transmisión en directo, debería verla aparecer en el reproductor de Brightcove que creó en los pasos anteriores.
