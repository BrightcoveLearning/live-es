Notificaciones del programador

Al crear flujos de trabajo con el programador, opcionalmente puede configurar notificaciones. Después de completar una tarea, el programador enviará una notificación al punto final configurado mediante el método http CORREO con detalles sobre qué acción se realizó y su éxito / fracaso. También puede configurar una notificación previa de una próxima tarea. Consulte la referencia de la API para configurar las notificaciones para su tipo de flujo de trabajo.

Cuerpo de notificación

Cuando el servidor envía una notificación a su notification_url, la solicitud incluirá un cuerpo JSON

Campo Descripción
acción Nombre de la acción. Ver Comportamiento
flujo de trabajo Flujo de trabajo actualizado después de la finalización de la acción. La estructura dependerá del tipo de flujo de trabajo.
error Si se produjo un error al completar la acción, este campo contendrá el error. el valor depende del tipo de acción.

Comportamiento

Nombre Tipo de flujo de trabajo Descripción
notificar-activar trabajo Aviso de alerta de que su trabajo se activará pronto.
activar trabajo Activación de trabajo completa. Si "error" está presente, la activación ha fallado y la desactivación se cancelará.
notificar-desactivar trabajo Aviso de alerta de que su trabajo se activará pronto.
desactivar trabajo Obligatorio la desactivación del trabajo. Si "error" está presente, la desactivación ha fallado.
notificar-clip acortar Aviso de alerta de que pronto se solicitará su clip.
acortar acortar Solicitud de clip completa. Si "error" está presente, la solicitud ha fallado.
