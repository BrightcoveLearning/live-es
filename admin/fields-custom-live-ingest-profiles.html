Campos para perfiles personalizados de ingesta en vivo

Este tema proporciona una referencia para los campos utilizados en los perfiles de ingesta para Live.

Campos de perfil

Propiedad Tipo Descripción Defecto
account_id entero los Video Cloud ID de la cuenta ninguno
name cuerda El nombre del perfil: debe ser único dentro de la cuenta. ninguno
display_name cuerda El nombre para mostrar para que el perfil aparezca en la página Administrador de Studio, puede ser el mismo que name, pero debe ser descriptivo para ayudar a los usuarios a comprender el uso previsto para el perfil ninguno
description cuerda La descripción del perfil ninguno
model_version entero Establecer esto en 1 para que el perfil se muestre en el módulo en vivo ninguno
renditions formación Matriz de objetos de interpretación []
packages formación Matriz de paquetes DRM: actualmente, DRM no es compatible, por lo que siempre debe ser una matriz vacía []

Campos de reproducción

Las tablas siguientes muestran campos que se pueden usar en el cuerpo de la solicitud para actualizar los perfiles de ingesta en vivo.

Propiedad Tipo Descripción Defecto
audio_bitrate entero tasa de bits de audio en kbps (mínimo: 16, máximo: 1024) fuente de coincidencia
audio_codec cuerda Códec de audio - aac | mp3 Determinado por el formato o perfil <- en la mayoría de los casos, aac será mejor / td>
decoder_bitrate_cap entero en kbps, la tasa de bits máxima alimentada al decodificador a través de un búfer ninguno
decoder_buffer_size entero en kbps, el tamaño del búfer alimentado al decodificador cuando se utiliza un bitrate_cap ninguno
forced_keyframe_rate número Fuerza un fotograma clave cada X fotogramas, pero aún permite fotogramas clave adicionales para cortar la escena, etc. ninguno
format cuerda Formato de vídeo - ts, mp4, flv ninguno
h264_profile cuerda H.264 tiene tres perfiles de uso común: Baseline(más bajo) y Main High.[1-5] baseline
hls_optimized_ts booleano Segmentos de tiempo optimizados para HLS: solo se aplica a la salida de HLS ninguno
keyframe_interval entero Número máximo de fotogramas entre fotogramas clave 250
keyframe_rate flotador Máximo de fotogramas clave por segundo keyframe_interval de 250
label cuerda Tipo de imagen para representaciones de imágenes - poster o thumbnail ninguno
live_stream booleano si esto será una transmisión de video en vivo ninguno
live_dvr_sliding_window_duration entero Duración de la transmisión para mantener disponible para la entrega de LiveDVR en segundos 300
max_video_bitrate entero en kbps, 123 = 123 kbps - solo h264 ninguno
max_frame_rate flotador limita el frame_rate en lugar de configurarlo. Debe establecerse en 30 a menos que su cuenta haya sido habilitada por Soporte para 60 FPS ninguno
media_type cuerda video o audio ninguno
video_bitrate entero en kbps, 123 = 123 kbps (mínimo: 64, máximo: 10000) ninguno
video_codec cuerda por ejemplo, h264, ts Determinado por el formato o perfil
width entero ancho del marco en píxeles ninguno
height entero altura del fotograma en píxeles ninguno
